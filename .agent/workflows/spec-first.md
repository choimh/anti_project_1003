# Spec-First 개발 방법론

코드를 작성하기 전에 다음 단계를 따릅니다:

## 1. 스펙 문서 작성 (`spec.md`)
프로젝트 루트에 `spec.md` 파일을 생성하고 아래 템플릿을 사용합니다.

## 2. 스펙 검토 및 승인
- 사용자와 함께 스펙 문서를 검토합니다
- 필요시 수정하고 최종 승인을 받습니다

## 3. 구현 계획 수립
- 승인된 스펙을 바탕으로 `implementation_plan.md`를 작성합니다
- 단계별 작업 항목을 정의합니다

## 4. 개발 진행
- 스펙과 계획에 따라 코드를 작성합니다
- 각 기능 완료 시 스펙 대비 검증합니다

## 5. 검증 및 완료
- 모든 스펙 요구사항이 충족되었는지 확인합니다
- `walkthrough.md`에 결과를 문서화합니다
